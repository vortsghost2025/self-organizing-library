2. Security Review  
2.1 Attestation (AttestationSupport.js)  
Issue	Impact	Recommendation
Algorithm string manipulation – crypto.createVerify(algorithm.toUpperCase().replace('-', '')) turns "rsa-sha256" into "RSASHA256", which is not a valid OpenSSL identifier. Node expects "RSA-SHA256" (or "RSA-SHA256" after simple toUpperCase()).	Verification will throw ERR_OSSL_EVP_UNSUPPORTED in production, silently returning valid:false.	Replace with crypto.createVerify(algorithm.toUpperCase()) or maintain a whitelist mapping ({ 'rsa-sha256': 'RSA-SHA256' }).
No canonicalisation of payload – JSON.stringify(payload) depends on property order. A different ordering between signer and verifier leads to a false‑negative.	Potential false‑negative verification; attacks could exploit ordering to produce a valid signature that the verifier rejects.	Serialize deterministically (e.g., stableStringify or custom ordered key array) before signing and verifying.
Public‑key loading assumes PEM format – the code returns the raw string but does not validate it. A malformed key will crash verifier.verify.	Runtime exception, denial‑of‑service.	Validate the PEM with a simple regex (/^-----BEGIN PUBLIC KEY-----/) and catch parsing errors, returning a structured INVALID_KEY result.
Trust‑store path is hard‑coded – S:/Archivist-Agent/.trust/keys.json.	Reduces configurability; different environments (dev/ci) cannot override.	Accept trustStorePath via constructor or read from an environment variable (process.env.ATTESTATION_TRUST_STORE).
Migration‑mode handling – mode 'hmac-only' forces valid:true for RSA signatures. This masks genuine verification failures.	In dual mode the system may accept forged RSA signatures when the mode is mistakenly set to 'hmac-only'.	Return a distinct status (MIGRATION_HMAC_ONLY) and a valid:false flag, letting callers decide whether to accept the artifact.
Trust‑store schema versioning – the code does not check trustStore.version.	Future schema changes could break verification silently.	Guard against missing keys or version mismatches and emit SCHEMA_MISMATCH with expectedVersion.
2.2 NexusGraph API (src/app/api/search/route.ts)  
* No input sanitisation – limit is parsed with parseInt but not bounded. A malicious client could request an excessively large limit (Number.MAX_SAFE_INTEGER) causing memory pressure.  
  Add a ceiling (Math.min(limit, 200)).  
* Potential timing‑side‑channel – Returning the raw search results without pagination hints can leak internal document counts. Consider adding a total field and capping per‑page results.
2.3 SwarmMind Agent Framework  
* Task validation – validateTask checks only for dangerous prototype keys and string length. However, it does not validate numeric fields, arrays, or nested objects. An attacker could embed a crafted object containing a function reference that later gets eval‑ed (future extensions).  
  Recommendation: Deep‑clone and whitelist allowed properties, or use a schema (e.g., zod) to enforce shape.  
* Trace log growth – The logTrace method caps the array size, which is good. Still, the trace entries contain the full task object; if a task includes large payloads this can bloat memory. Store only a hash or a trimmed version ({ id, type, goal }).  
* No concurrency guard – Multiple simultaneous execute calls on the same agent could interleave currentTask state. Either make execute reject when status !== 'idle' or queue tasks.

3. Performance Review  
Module	Observation
src/lib/db.ts – getDb lazily imports @/db on each call. The first call incurs a dynamic import(). Subsequent calls are cheap, but if many concurrent operations start before the first import resolves they will each trigger the import.	Initialise the DB connection at process start (bun start) and export a singleton.
Search (searchDocuments) – Uses LIKE '%query%' for both title and content. This performs a full table scan on SQLite, which is acceptable for a demo but will degrade with >5 k docs.	Create a full‑text search virtual table (FTS5) and query via MATCH.
Mock data generators – getMockDocuments builds 50‑100 objects on each request, each with new Date() calls, random colours, etc.	Cache the mock dataset for the duration of a request (or reuse a static fixture) to reduce CPU churn during demo mode.
SwarmMind processTask – Uses await new Promise(r=>setTimeout(r, workTime)) to simulate work. No real CPU impact, but the workTime values are hard‑coded. If a real task runs a heavy computation, the event loop may be blocked.	Offload CPU‑heavy work to a worker thread or child process; keep the main agent loop non‑blocking.
Attestation verification – crypto.createVerify and verifier.verify are synchronous. For a high‑throughput service this could become a bottleneck.	Use the asynchronous crypto.verify API (crypto.verify(null, data, key, signature, callback)) or move verification to a worker pool.

```javascript
function stableStringify(value) {
  if (value === null || typeof value !== 'object') {
    return JSON.stringify(value);
  }

  if (Array.isArray(value)) {
    return `[${value.map(stableStringify).join(',')}]`;
  }

  const keys = Object.keys(value).sort();
  const entries = keys.map((k) => `${JSON.stringify(k)}:${stableStringify(value[k])}`);
  return `{${entries.join(',')}}`;
}

module.exports = { stableStringify };
```

## C. Archivist: harden `TrustStoreManager.js`

Replace the current single-key overwrite behavior with versioned key enforcement.

```javascript
const fs = require('fs');
const path = require('path');
const crypto = require('crypto');
const {
  TRUST_STORE_PATH,
  TRUST_STORE_VERSION,
  VERIFY_REASON
} = require('./constants');

class TrustStoreManager {
  constructor(options = {}) {
    this.trustStorePath = options.trustStorePath || TRUST_STORE_PATH;
    this.trustStore = null;
    this._load();
  }

  _load() {
    if (!fs.existsSync(this.trustStorePath)) {
      throw new Error(`${VERIFY_REASON.TRUST_STORE_MISSING}: ${this.trustStorePath}`);
    }
    this.trustStore = JSON.parse(fs.readFileSync(this.trustStorePath, 'utf8'));
    if (this.trustStore.version !== TRUST_STORE_VERSION || !this.trustStore.keys) {
      throw new Error(VERIFY_REASON.SCHEMA_MISMATCH);
    }
  }

  _save() {
    this.trustStore.updated_at = new Date().toISOString();
    fs.writeFileSync(this.trustStorePath, JSON.stringify(this.trustStore, null, 2), 'utf8');
  }

  _validatePem(publicKeyPem) {
    if (!/^-----BEGIN PUBLIC KEY-----[\s\S]+-----END PUBLIC KEY-----\s*$/.test(publicKeyPem)) {
      throw new Error(VERIFY_REASON.INVALID_KEY);
    }
    crypto.createPublicKey(publicKeyPem);
  }

  registerKey(laneId, publicKeyPem, keyId) {
    const entry = this.trustStore.keys[laneId];
    if (!entry) throw new Error(VERIFY_REASON.UNKNOWN_LANE);
    if (entry.revoked_at) throw new Error(VERIFY_REASON.KEY_REVOKED);

    this._validatePem(publicKeyPem);

    this.trustStore.keys[laneId] = {
      ...entry,
      lane_id: laneId,
      public_key_pem: publicKeyPem,
      key_id: keyId,
      algorithm: 'RS256',
      registered_at: new Date().toISOString(),
      revoked_at: null
    };

    this._save();
    return this.trustStore.keys[laneId];
  }

  getKey(laneId) {
    return this.trustStore.keys[laneId] || null;
  }

  revokeKey(laneId, reason = 'Key compromised') {
    const entry = this.trustStore.keys[laneId];
    if (!entry) throw new Error(VERIFY_REASON.UNKNOWN_LANE);

    entry.revoked_at = new Date().toISOString();
    entry.revocation_reason = reason;
    this._save();
    return entry;
  }
}

module.exports = { TrustStoreManager };
```

## D. SwarmMind + Archivist: unified `Verifier.js`

Use this structure in both SwarmMind and Archivist.

```javascript
const fs = require('fs');
const crypto = require('crypto');
const {
  TRUST_STORE_PATH,
  TRUST_STORE_VERSION,
  MIGRATION_MODE,
  VERIFY_REASON,
  ALG_MAP
} = require('./constants');
const { stableStringify } = require('./stableStringify');

class Verifier {
  constructor(options = {}) {
    this.trustStorePath = options.trustStorePath || TRUST_STORE_PATH;
    this.migrationMode = options.migrationMode || MIGRATION_MODE.DUAL;
    this.trustStore = null;
    this._loadTrustStore();
  }

  _loadTrustStore() {
    if (!fs.existsSync(this.trustStorePath)) {
      this.trustStore = null;
      return;
    }
    this.trustStore = JSON.parse(fs.readFileSync(this.trustStorePath, 'utf8'));
  }

  _checkSchema() {
    if (!this.trustStore) {
      return { ok: false, reason: VERIFY_REASON.TRUST_STORE_MISSING };
    }
    if (this.trustStore.version !== TRUST_STORE_VERSION || !this.trustStore.keys) {
      return { ok: false, reason: VERIFY_REASON.SCHEMA_MISMATCH };
    }
    return { ok: true };
  }

  getPublicKeyEntry(laneId) {
    const schema = this._checkSchema();
    if (!schema.ok) return { error: schema.reason };

    const entry = this.trustStore.keys[laneId];
    if (!entry) return { error: VERIFY_REASON.UNKNOWN_LANE };
    if (entry.revoked_at) return { error: VERIFY_REASON.KEY_REVOKED };
    if (!entry.public_key_pem) return { error: VERIFY_REASON.KEY_NOT_FOUND };

    try {
      crypto.createPublicKey(entry.public_key_pem);
    } catch {
      return { error: VERIFY_REASON.INVALID_KEY };
    }

    return { entry };
  }

  verifyPayload(payload, signature, laneId, algorithm = 'rsa-sha256', keyId = null) {
    const alg = ALG_MAP[algorithm] || ALG_MAP[algorithm?.toUpperCase?.()];
    if (!alg) {
      return { valid: false, reason: VERIFY_REASON.UNSUPPORTED_ALGORITHM, lane: laneId };
    }

    const keyResult = this.getPublicKeyEntry(laneId);
    if (keyResult.error) {
      return { valid: false, reason: keyResult.error, lane: laneId };
    }

    const entry = keyResult.entry;
    if (keyId && entry.key_id && keyId !== entry.key_id) {
      return { valid: false, reason: VERIFY_REASON.KEY_ID_MISMATCH, lane: laneId };
    }

    try {
      const verifier = crypto.createVerify(alg);
      verifier.update(stableStringify(payload));
      verifier.end();

      const valid = verifier.verify(entry.public_key_pem, signature, 'base64');

      return {
        valid,
        reason: valid ? VERIFY_REASON.VERIFIED : VERIFY_REASON.SIGNATURE_MISMATCH,
        lane: laneId,
        algorithm
      };
    } catch (error) {
      return {
        valid: false,
        reason: VERIFY_REASON.VERIFICATION_ERROR,
        lane: laneId,
        error: error.message
      };
    }
  }

  verifyQueueItem(item) {
    if (!item.signature) {
      return {
        valid: false,
        reason: VERIFY_REASON.MISSING_SIGNATURE,
        lane: item.origin_lane || null
      };
    }

    const signedLane = item.payload?.lane || item.attestation?.lane || item.origin_lane;
    if (!item.origin_lane || signedLane !== item.origin_lane) {
      return {
        valid: false,
        reason: VERIFY_REASON.LANE_MISMATCH,
        lane: item.origin_lane || null
      };
    }

    const signablePayload = {
      id: item.id,
      timestamp: item.timestamp,
      origin_lane: item.origin_lane,
      target_lane: item.target_lane,
      type: item.type,
      artifact_path: item.artifact_path,
      required_action: item.required_action,
      payload: item.payload
    };

    return this.verifyPayload(
      signablePayload,
      item.signature,
      item.origin_lane,
      item.algorithm || 'rsa-sha256',
      item.key_id || null
    );
  }
}

module.exports = { Verifier };
```
