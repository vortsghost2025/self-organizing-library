## 2. Write-time Schema Validator

You already have messaging. Now make invalid writes impossible.

### Purpose

Catch bad messages **before** they hit inboxes.

### Minimal scope

Validate at write time:

* required fields present
* canonical `from_lane` / `to_lane`
* valid schema version
* idempotency key present
* lease block present
* evidence block present
* no write to non-canonical path

### Enforcement rule

If validation fails:

* write is rejected
* error logged
* message not emitted

### Don’t do yet

* don’t build a universal framework
* don’t add remote endpoints
* don’t over-generalize versions

Just validate the format you already use.

---

## 3. Delivery Proof Artifact

Stop saying “delivered” unless it can be proven.

### Purpose

Turn delivery from a claim into an artifact.

### File shape

```json
{
  "type": "delivery_proof",
  "message_id": "2026-04-20T23-10-00Z_swarmmind_codex-audit-response",
  "from_lane": "swarmmind",
  "to_lane": "archivist",
  "canonical_path": "S:/Archivist-Agent/lanes/archivist/inbox/2026-04-20T23-10-00Z_swarmmind_codex-audit-response.json",
  "payload_hash": "sha256:...",
  "delivered": true,
  "verified_at": "2026-04-21T00:00:00Z"
}
```

### Rule

A message is only “delivered” if:

* file exists at canonical target path
* payload hash matches
* proof artifact exists

This fits your current lattice directly.

# Phase 2 — improve observability without becoming the router again

## 4. Observer Summary Artifact

Not another full lane. Just a summary writer.

### Purpose

Give you back the “alive” feeling without turning you back into the system.

### Output

Every 60–120 seconds, emit:

```json
{
  "timestamp": "...",
  "lanes": {
    "archivist": { "heartbeat_age_s": 20, "processed_recent": 1 },
    "library": { "heartbeat_age_s": 15, "processed_recent": 2 },
    "swarmmind": { "heartbeat_age_s": 18, "processed_recent": 0 },
    "kernel": { "heartbeat_age_s": 12, "processed_recent": 1 }
  },
  "events": [
    "library processed 1 message",
    "swarmmind wrote delivery proof",
    "kernel profiling still running"
  ]
}
```

### Rule

This is read-only observation.
No authority. No decisions. No routing.

## 5. Coordination Freshness vs Liveness

Keep your correction.

### Rule

Never call a lane “dead” from:

* stale heartbeat
* stale commit
* lack of recent artifact

Only report:

* fresh

* no recent signal

That correction was one of the strongest things your system learned.
