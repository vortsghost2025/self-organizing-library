Why the Lattice is "Leaking" (The 3 Likely Culprits)

1. The "Separate Process" Loophole (Process Isolation Failure)
Your LaneContextGate uses a global fs monkey-patch. In Node.js, a monkey-patch only exists within the memory space of that specific PID (Process ID).

The Bug: If an agent spawns a child process (e.g., exec('node script.js') or sh script.sh), the child process does not inherit the monkey-patch.

The Bypass: The agent (Identity) might be constrained, but the sub-shell it spawns is "blind" to the gate and writes directly to the disk, bypassing the lattice entirely.

Fix: You must wrap the execution layer in an environment variable check (e.g., NODE_OPTIONS='--require ./src/core/laneContextGate.js') to force the gate into every sub-process.

3. Asynchronous Race Conditions (The "Write-Before-Gate" Race)
If your agents use certain native C++ bindings or fs.promises that aren't fully covered by the monkey-patch, they might find a "raw" path to the disk.
