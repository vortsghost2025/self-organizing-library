Gate Behavior
LaneContextGate enforces policy require_authority_100_or_same_lane:
1. Session start (governed-start.js): Load ownership registry, verify session-lock lane, reconcile pwd vs session — abort on mismatch
2. Pre-write hook (guardedWriteFile in resolve-governance-v2.js): Every file write checks target file's owning lane against session lane/authority
3. Same-lane write → ALLOWED
4. Cross-lane write with authority < 100 → BLOCKED + HOLD + operator resolution required
5. Hold state (enterHold/exitHold): All further writes blocked until operator clears
---
Enforcement Points
Point	Implementation
Session Start	governed-start.js initializes gate before any operation
Pre-write Hook	guardedWriteFile() wraps all fs.writeFileSync in resolve-governance-v2.js
Directory Change	Lane detection via determineLaneFromPath() for every target
Registry Update	Gate blocks writes to SESSION_REGISTRY.json (owned by archivist-agent) from SwarmMind (80)
